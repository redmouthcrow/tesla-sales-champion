# 产品需求文档 (Product Specification)

> **文档性质说明**：本文档描述的是 **Agent Skill 规格**（`SKILL.md` + 内置知识文件），供微信小程序后端组装进 DeepSeek-flash 的 system prompt 使用。**不包含**小程序工程、后端 API、会话存储、性能指标等产品化内容——那些由宿主应用自行实现。

## 项目概述

**项目名称**：特斯拉客户答疑 Skill（Tesla Sales Champion Q&A Skill）

**一句话描述**：一个内置培训文档与销冠 Q&A 的 Agent Skill，使大模型在回答特斯拉客户问题时口径准确、支持多轮追问、对易变政策标注时效，且不谈单。

**目标**：
- 将 **B 培训文档**（内置、可更新）与 **C 销冠 Q&A** 蒸馏为 Skill 可引用的知识
- 约束模型：**只答疑，不推销、不逼单**
- 支持宿主传入对话历史时的**多轮连贯回答**
- 对价格/补贴/金融等易变信息**强制时效提示**

**宿主集成方式（仅说明边界，非本文档交付物）**：
- 微信小程序将用户问题 + 会话历史交给后端，后端加载本 Skill 后调用 **DeepSeek-flash**
- Skill 不感知微信 API，不管理 `session_id`

## 目标用户

**答复对象**：通过宿主小程序提问的**终端客户**（C 端）。

**Skill 使用者**：组装 prompt 的**后端开发者**（将 `SKILL.md` 与 `knowledge/` 注入模型请求）。

**用户特征（影响措辞，非功能需求）**：
- 普通微信用户，简体中文
- 只想获得客观解答，不愿被推销

## 核心功能

以下功能均为 **Skill 内行为规则**，由模型在单次/多轮生成中执行，非独立微服务。

### 功能 1：单轮答疑

**功能描述**：根据用户当前问题与检索到的知识片段，生成面向客户的答复。

**输入**（由宿主传入模型请求）：
- 当前用户消息
- 可选：检索到的 `knowledge/` 片段

**输出**（模型生成）：
- 客户可读的中文答复
- 知识不足时触发功能 4

**业务规则**：
- 只答特斯拉相关：产品、购车、用车、充电、售后政策等
- **禁止**主动推销、逼单、限时压迫、引导加销售微信
- 不得编造知识库中不存在的事实性政策/价格数字

**异常处理**：
- 问题为空：提示用户具体描述问题
- 与特斯拉无关：礼貌说明只能回答特斯拉相关问题

**优先级**：高

**状态**：已定义（见 `SKILL.md`）

---

### 功能 2：多轮上下文追问

**功能描述**：当宿主将历史对话一并传入时，理解指代与追问，保持话题连贯。

**输入**：
- 当前用户消息
- `conversation_history`（宿主维护并传入，格式由宿主定义）

**输出**：
- 承接上下文的答复

**业务规则**：
- 以**最新明确意图**为准；若与早期话题冲突，简要说明后回答新问题
- 不得用历史对话臆测补全知识库中不存在的内容
- 上下文截断策略由**宿主**控制，Skill 不规定轮数或超时

**异常处理**：
- 无历史记录时：按单轮答疑处理

**优先级**：高

**状态**：已定义（见 `SKILL.md`）

---

### 功能 3：政策/价格时效标注

**功能描述**：涉及价格、补贴、金融、促销、交付周期等易变信息时，答复中必须带时效与免责表述。

**输入**：
- 用户问题（含价格/优惠/贷款等意图）

**输出**：
- 正文答复 + 不可省略的时效/免责句

**业务规则**：
- 命中易变话题时，必须包含类似表述：「信息截至 [日期/版本]，最终以官方或门店公示为准」「不构成要约或承诺」
- 客户议价类问题（「还能便宜吗」）：说明无法在此承诺优惠，引导以门店当时政策为准
- 知识条目中若有 `effective_date` / 文档版本，须在答复中体现

**异常处理**：
- 知识无日期：使用「请以官方最新公示为准」模板

**优先级**：高

**状态**：已定义（见 `SKILL.md`）

---

### 功能 4：无依据时明确拒答

**功能描述**：知识库无法支撑时，明确说明无法确认，不幻觉作答。

**输入**：
- 用户问题 + 检索结果（空或不足以回答）

**输出**：
- 拒答说明 + 建议通过官方/门店渠道确认（具体渠道文案可写在知识库）

**业务规则**：
- 禁止用模型通识「猜」特斯拉专属政策数字
- 语气礼貌、简短

**异常处理**：无

**优先级**：中

**状态**：已定义（见 `SKILL.md`）

---

### 功能 5：引用溯源（弱化）

**功能描述**：内部可追溯依据；对客户答复**不强制**展示引用列表。

**输入**：检索命中的知识条目 ID/标题

**输出**：答复正文（引用仅在有把握时简短提及「参考培训材料/销冠经验」即可）

**业务规则**：v1 不作为阻塞项

**优先级**：低

**状态**：已定义（见 `SKILL.md`，弱化）

## 功能优先级

### 高优先级
1. 多轮上下文追问（F2）
2. 政策/价格时效标注（F3）
3. 单轮答疑（F1）

### 中优先级
1. 无依据时明确拒答（F4）

### 低优先级
1. 引用溯源（F5，原 F6 重新编号）

### 明确不做
- 销售谈单、异议攻防、逼单教练
- 合规章节 / 违规词过滤清单（F5 已删除）
- 小程序 UI、会话存储、API 设计、SLA、成功指标

## 知识库规范（Skill 交付物核心）

### B：培训文档

| 项 | 规则 |
|----|------|
| 存放 | 内置在 Skill 仓库 `knowledge/training/`（或等价路径） |
| 版本 | 始终维护**最新版**一份；不保留多版本并行 |
| 更新 | **Admin 手动替换文件**后发布 Skill 版本即可，无自动爬取 |
| 格式 | 由运营提供，实现期转为 Markdown/纯文本供模型引用 |

### C：销冠经验

| 项 | 规则 |
|----|------|
| 形态 | 整理为 **Q&A** 文件（`knowledge/champion-qa/`） |
| 冲突 | **以 C（销冠 Q&A）为准** 作为答复主口径 |
| 备注 | 若与 B 不一致，答复中须**备注 B 的表述**（例如：「培训材料中写的是…，实际工作中更常见的是…」） |
| 禁止 | C 不得凭空捏造 B 中完全不存在的官方政策条文；无 B 支撑的政策数字仍须拒答或标注不确定 |

### 知识检索（建议实现，非宿主职责）

- Skill 实现者可选用：全文注入（小库）或 RAG；具体技术在 `SKILL.md` 实现说明中写清
- 检索与注入对**宿主透明**——宿主只需加载 Skill 并按其说明组装请求

## AI增强功能

本 Skill **即** AI 增强载体；以下为 Skill 内嵌的模型行为指令。

### AI增强 1：问题类型识别

**AI能力描述**：识别问题是否涉及 `pricing` / `promotion` / `finance` / `delivery`，以触发 F3 时效块。

**触发条件**：每条用户消息

**输入**：用户消息 + 可选历史

**输出**：答复结构（含或不含时效块）

**示例**：「现在有什么补贴」→ 强制附带时效与免责句

---

### AI增强 2：多意图拆解

**AI能力描述**：一条消息含多个问题时，分点作答。

**触发条件**：检测到 ≥2 个独立意图

**输入**：用户消息

**输出**：分点编号的 `answer`

**示例**：「续航多少、桩怎么装、首付几成」→ 三点分别回答

---
