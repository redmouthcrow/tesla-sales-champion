## [1.1.0] - 2026-05-20

### 变更类型：修改 / 删除 / 重构

### 修改功能
- **文档定位**：从「完整小程序产品」收窄为 **Agent Skill 规格**；小程序/后端/SLA/成功指标移出范围
- **知识库 B**：明确内置最新版、admin 手动更新
- **知识库 C**：明确 Q&A 形态；冲突规则由「B 优先」改为 **C 优先并备注 B 内容**
- **功能编号**：原 F6 引用溯源重编号为 F5；删除原 F5 合规过滤

### 删除功能
- **合规过滤（原 F5）**：产品方明确不需要合规章节，整条删除
- **非功能性需求（性能/安全/并发）**：不属于 Skill 交付范围
- **待确认项 T4–T6**：不再追踪（会话、指标、并发由宿主负责）

### 功能调整
- 多轮会话轮数、超时：从 Skill 规格移除，改由宿主实现

### 影响范围
- **实现范围**：仅需交付 `SKILL.md` + `knowledge/`，无需按原 Spec 建设独立后端功能模块
- **宿主集成**：宿主自行负责 `session_id`、历史消息、API Key、DeepSeek 调用

### 兼容性说明
- 向后兼容：对 Skill 消费者无 API 破坏（首版 Skill 尚未发布）
- 迁移指南：若已按 1.0.0 开发合规过滤模块，**删除**该模块

## [1.0.0] - 2026-05-20

### 变更类型：新增

### 新增功能
- **单轮客户问答（F1）**：C 端客户在小程序内提问，获得基于知识库的答复
- **多轮上下文追问（F2）**：基于 `session_id` 与 `conversation_history` 的连续对话
- **政策/价格时效标注（F3）**：易变信息强制时效提示与免责声明
- **无依据时明确拒答（F4）**：低置信度或低检索分时拒答并引导官方渠道
- **合规过滤（F5）**：后置过滤违规承诺与不当表述
- **引用溯源（F6）**：后台记录依据来源，对客户默认不展示

### 新增 AI 能力
- **问题类型自动分类**：路由时效与检索策略
- **复杂问题拆解**：多意图分点作答
- **知识库检索增强（RAG）**：B 培训文档 + C 销冠经验片段注入

### 非功能性需求调整
- 明确性能、安全、可用性、微信小程序兼容性初版要求
- API Key 服务端保管，禁止小程序前端直连模型

### 技术栈调整
- 确定主力模型为 **DeepSeek-flash**
- 确定交付形态为 **微信小程序 + 独立后端 + Agent Skill**

### 影响范围
- **全部功能**：初始版本，无历史兼容负担
- **范围外能力**：销售谈单、推销、CRM 等均不在本版本影响面

### 兼容性说明
- 向后兼容：不适用（首版）
- 迁移指南：无
- 备注：附录 T1–T6 待确认项闭环后建议发布 1.1.0

## 版本号规范

遵循语义化版本规范：`MAJOR.MINOR.PATCH`

- **MAJOR（主版本号）**：不兼容的 API 修改或重大功能变更
- **MINOR（次版本号）**：向下兼容的功能性新增
- **PATCH（修订号）**：向下兼容的问题修正

### 修改（Changed）
